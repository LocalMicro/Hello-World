<p class="ListParagraph tm24"><span class="tm23"></span><span class="tm25">&#61623; </span><span class="tm23">Hardware</span></p>
<p class="ListParagraph tm26"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">This runs on V1 of the Kinect for 360. Model numbers 1414, 1473 and maybe 1517 (Kinect for Windows) but I have not tried
that one. I tried the other 1.</span></p>
<p class="Normal tm22"><span class="tm23"></span><strong><span class="tm28">Windows 10</span></strong><span class="tm23"><br />You need to download</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">the </span><a href="https://www.microsoft.com/en-us/download/details.aspx?id=48145"><span
 class="tm31">Visual C++ run-time components </span></a><span class="tm23">,</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - Download Visual C++ the 2013 release for x86 and x64 bit from the microsoft website: </span><u><a
 href="https://www.microsoft.com/en-us/download/details.aspx?id=40784"><span class="tm31">https://www.microsoft.com/en-us/download/details.aspx?id=40784</span></a></u></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">You do not need the &#8220;arm version.  </span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">Earlier and later releases do not have the right &#8220;vcomp&#8221; file.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">Without this the application complains it does not have &#8220;vcomp120.dll&#8221;</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">the Free USB Driver tool: </span><a
 href="http://zadig.akeo.ie/"><span class="tm31">Zadig</span></a><span class="tm23"> and</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC &#8211; This is OK</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">the Magic Sand software it-self</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - This is OK</span></p>
<p class="Normal tm21"><strong><span class="tm16">Quick start</span></strong></p>
<p class="Normal tm22"><span class="tm23">Download and unzip the files wherever you want.</span></p>
<p class="Normal tm21"><strong><span class="tm34">Windows</span></strong></p>
<p class="Normal tm29"><span class="tm23">1. Update your video drivers.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - Use Device Manager</span></p>
<p class="Normal tm29"><span class="tm23">2. Install the Visual C++ run-time components (in particular the x86 component even if you run on a 64bits Windows).</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - As mentioned, 2013 version</span></p>
<p class="Normal tm29"><span class="tm23">3. Install the Kinect drivers with the Free USB Driver tool Zadig:</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">Select your Xbox Camera, Xbox Audio and Xbox Motor in the drop down and then select the libusb-win32( v1.2.6.0) driver from the
driver menu (click on the up/down arrows) and click install.</span></p>
<p class="Normal tm35"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC -Zadig screen</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23"></span>
<img src="AR Sandbox for Windows_files/img00002.PNG" width="312" height="138" alt="" /><span class="tm23"><br />You need to do this operation </span><strong><span
 class="tm28">two to three times</span></strong><span class="tm23">: once for the camera, once for motor and once for the audio. </span></p>
<p class="Normal tm35"><span class="tm23"></span><span class="tm27">o </span><span class="tm23"></span>NC - <span class="tm23">We did not see the motor but had a security item instead so we installed the same driver for that.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">See the </span><a href="https://github.com/OpenKinect/libfreenect"><span
 class="tm31">freenect github</span></a><span class="tm23"> and </span><a href="https://github.com/openframeworks/openFrameworks/tree/master/addons/ofxKinect"><span
 class="tm31">ofxKinect github</span></a><span class="tm23"> for more information.<br />If there is a problem with the detection of the Kinect, you can install the </span><a
 href="https://www.microsoft.com/en-us/download/details.aspx?id=28782"><span class="tm31">Kinect V1 (SDK 1.0)</span></a><span
 class="tm23"> and run Zadig again.</span></p>
<p class="Normal tm35"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - We installed the Kinect V1 SDK because we did not see the Motor listed as USB device in Zadig or on Device Manager (even
in &#8220;Other&#8221;). We only saw Kinect Audio and Video and no Motor but a Security in addition. So we ran Zadig and installed the USB driver </span><strong><em><span
 class="tm36">lbusb-win32(v1.2.6.0)</span></em></strong><span class="tm23"> on all 3 devices we saw.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">The drivers will identify different deices in the Kinect depending on model 1414 or 1473. That does not matter, all that matters
is that you assign all of the Kinect deices to the libusb-win32( v1.2.6.0)  driver.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">Kinect model 1517 I have not<a
 id="aGoBack"></a> verified.</span></p>
<p class="Normal tm21"><strong><span class="tm34">Setting up the system</span></strong></p>
<p class="Normal tm22"><span class="tm23">Connect and turn on the projector and the Kinect and start the Magic-Sand software.</span></p>
<p class="Normal tm22"><span class="tm23">By default the software starts in a </span><strong><span class="tm28">setup</span></strong><span
 class="tm23"> mode where the depth or color image from the Kinect can be seen in the user interface and the projector projects a completely white image. This way it is easy to check if the Kinect is running (on Windows 10
machines it can be necessary to plug and unplug the Kinect several times before it starts) and if the projector is working. The status of the Kinect and the projector can be seen in the status window to the lower left in the
user interface.</span></p>
<p class="Normal tm22"><span class="tm23">In </span><strong><span class="tm28">setup</span></strong><span class="tm23"> mode the physical positions of the Kinect and projector can be optimized.</span></p>
<p class="ListParagraph tm24"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - After getting past the missing vcomp driver (this took a long time to figure out) we saw the projection go white and
had trouble seeing anything else until we configured the PC display with a monitor second screen extension feeding the projector. At Bryce&#8217;s suggestion we configured the Projector screen above the PC screen. Then the
Sandbox output went on the wall (projector) and we could see the program other windows on the PC screen.</span></p>
<p class="ListParagraph tm24"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - There is a screen where you see the Kinect Camera view and have the program controls and also a command line window
that shows what is happening. This was useful to see errors and progress.</span></p>
<p class="Normal tm22"><span class="tm23">&#160;</span></p>
<p class="ListParagraph tm24"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - Physical Positions. After some experimenting with trying to get it to calibrate, we figured out that we needed to
get the camera to be a square onto the projected image as possible so we put the Kinect on a tall PC on a table  </span></p>
<p class="Normal tm22"><span class="tm23">&#160;</span></p>
<p class="Normal tm22"><span class="tm23">&#160;</span></p>
<p class="Normal tm21"><strong><span class="tm37">Calibration</span></strong></p>
<p class="Normal tm22"><span class="tm23">To calibrate the system so the Kinect and the projector is in correspondence a few steps are needed:</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Flatten the sand in the sand box.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - The projector wall image was flat</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Make sure that you see either the depth image or the color image from the Kinect (click </span><strong><span
 class="tm28">advanced|Display Kinect Depth View</span></strong><span class="tm23">)</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - This we could see on the PC window</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Press </span><strong><span
 class="tm28">Calibration|Manually Draw Sand Region</span></strong></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Define the sand region by drawing a rectangle with the mouse on the Kinect Depth or Color view</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - We tried smaller areas to try to compensate for PC low processing, but determined after seeing the calibration process that
we needed as much of the projected area as possible to be enclosed, as it uses the whole area for calibration.</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Press </span><strong><span
 class="tm28">Automatically Calibrate Kinect &amp; Projector</span></strong><span class="tm23"> - a series of Chessboard patterns are now projected on the sand.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - It will project a maybe 5x5 checkerboard on various places on the projected area then monitor it and subsequently show it
on the PC Kinect camera window before displaying the next one with the projector. It does this a lot and takes a while. You can see the successful operations complete on the command line window.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23"></span><span class="tm38"> </span><span
 class="tm23">
<img src="AR Sandbox for Windows_files/img00003.PNG" width="366" height="206" alt="" /></span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">When a prompt appears cover the sand box with a light piece of cardboard or similar.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - What it is doing is finding the range to the top of the sandbox, assuming that the sand level will not exceed this. Based
on this range and the level sand range determined during prior calibration activity, it can calculate the height allocation for the various sand height levels it will sense and subsequently illuminate with different colors.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - To give some effect of sandbox spacing between top of sandbox and sand level inside, we moved the Kinect camera about 4 inches
closer to the screen before pressing OK.</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Press ok - a series of Chessboard patterns are now projected on the cardboard/plate.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - It did this OK</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - We now had to move the camera back to the prior position.</span></p>
<p class="Normal tm22"><span class="tm23">If the calibration is successful the status window should be updated showing that all is ok.</span></p>
<p class="ListParagraph tm24"><span class="tm23"></span><span class="tm27">o </span><span class="tm23">NC - It was.</span></p>
<p class="Normal tm21"><strong><span class="tm28">Debug mode for calibration</span></strong></p>
<p class="Normal tm22"><span class="tm23">If the calibration was not successful a debug mode can be enabled that will place debug files in the </span><strong><span
 class="tm28">data\DebugFiles</span></strong><span class="tm23"> folder. These might point you in the direction of why the calibration failed. Do this by enabling </span><strong><span
 class="tm28">advanced|Dump Debug</span></strong><span class="tm23"> and run the calibration routine again.</span></p>
<p class="Normal tm22"><span class="tm23"></span><strong><span class="tm28">Tips for a nice calibration:</span></strong></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Turn off the light in the room for the calibration, depending on the lighting, it may take several tries to get it to complete
the calibration sometimes.</span></p>
<p class="Normal tm29"><span class="tm23"></span><span class="tm30">&#61623; </span><span class="tm23">Make sure you have a perfectly flat cover for the second part of the calibration. If it&#8217;s not flat, it may not calibrate
properly. Also, make sure the cover has a matte surface otherwise the Kinect may have problem to detect the depth.</span></p>
<p class="Normal tm21"><strong><span class="tm34">Starting the Application</span></strong></p>
<p class="Normal tm22"><span class="tm23">If the calibration was successful or if a calibration was done before, the application can be started by pressing space or pushing the </span><strong><span
 class="tm28">Run</span></strong><span class="tm23"> button.</span></p>
<p class="Normal tm22"><span class="tm23">Now a colored map with iso-lines should appear on the sand. The framerate should be close to 60 FPS for modern PCs.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - My PC showed about 10-15 FPS &#8211; lol.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - With this calibration, moving the camera closer to the wall will raise the effective sand height and change the displayed
colors. By moving it in and out and tilting it we could induce display of high ground and water.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - By introducing white object like paper held above the screen image, we could show effect of making hills.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - The Projected image would show the new sand height colors after about 1 second as this is a low performance PC.  </span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC - Ideally a delay of less than 1/10 second is needed for good user interface response.</span></p>
<p class="Normal tm32"><span class="tm23"></span><span class="tm33">o </span><span class="tm23">NC &#8211; Re-imaging the loaner LINUX laptop with Windows and using that would result in a much faster update rate and smoother
response.</span></p>
